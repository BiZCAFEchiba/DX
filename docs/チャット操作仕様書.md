# チャット操作仕様書（LINE WORKS）

## 概要

LINE WORKSのグループチャットでボットに話しかけることで、シフトの確認・変更・交代申請ができます。
すべての操作はグループチャット上で完結します（個人トークへの送信はありません）。

---

## 初回登録（受信ID登録）

チャット機能を使う前に、1回だけ自分のIDを登録する必要があります。

**登録手順**
1. 管理者がスプレッドシートから「受信ID取得」を実行
2. ボットから個人DM「フルネームを返信してください」が届く
3. スペースなしのフルネームをそのまま返信する

```
例）田中勇平
```

4. ボットが「✅ 田中勇平 さんのIDを登録しました！」と返答すれば完了

**登録後にできること**
- シフト確認・変更がチャットからできる
- 【確認しました】ボタンが正常に機能する

---

## シフト確認

### 起動キーワード

グループチャットで以下のどれかを含むメッセージを送ると開始します。

- 「シフト教えて」
- 「シフト照会」
- 「シフト見せて」

### 操作フロー

```
あなた：シフト教えて

ボット：何を確認しますか？

        1. 自分の今後のシフト
        2. 特定の日のシフト
        3. キャンセル

あなた：1

ボット：📋 田中勇平 さんの今後のシフト

        3月1日(日)　10:00〜18:00
        3月3日(火)　13:00〜20:00
        3月5日(木)　10:00〜15:00
```

```
あなた：2（を選択した場合）

ボット：何日のシフトを確認しますか？
        例）3/1

あなた：3/1

ボット：📋 3月1日(日) のシフト

        10:00〜18:00　田中勇平
        13:00〜20:00　星野竜大
        15:00〜20:00　杉本尚哉
```

### 日付の入力形式

以下の形式に対応しています。

| 形式 | 例 |
|------|----|
| 月/日 | 3/1 |
| 月月日 | 3月1日 |
| 年/月/日 | 2026/3/1 |
| 年-月-日 | 2026-3-1 |

## シフト変更

### 起動キーワード

グループチャットで「**シフト変更**」を含むメッセージを送ると開始します。

```
例：シフト変更したい
例：シフト変更お願いします
```

### 変更の種類

| 番号 | 内容 |
|------|------|
| 1 | 自分のシフト時間を変更（時間変更） |
| 2 | 誰かと交代（シフトスワップ） |
| 3 | まちがえました（キャンセル） |

数字は半角・全角どちらでも入力できます（1 または １）。

---

### 1. 自分のシフト時間変更

```
あなた：シフト変更

ボット：変更の種類を選んでください。

        1. 自分のシフト時間を変更
        2. 誰かと交代
        3. まちがえました（キャンセル）

あなた：1

ボット：何日のシフトを変更しますか？
        例）3/1

あなた：3/1

ボット：3月1日(日) のシフトを何時〜何時に変更しますか？
        例）15:30〜18:00

あなた：15:30〜18:00

ボット：以下の内容で変更申請します。よろしいですか？

        📅 3月1日(日)
        👤 田中勇平
        ⏱ 15:30〜18:00

        はい／いいえ

あなた：はい

ボット：✅ シフトを変更しました！

        📅 3月1日(日)
        👤 田中勇平
        ⏱ 15:30〜18:00
```

---

### 2. 交代申請

```
あなた：シフト変更

ボット：変更の種類を選んでください。
        （省略）

あなた：2

ボット：誰と交代しますか？
        相手の名前を入力してください。
        例）田中勇平

あなた：星野竜大

ボット：星野竜大 さんと交代ですね。
        何日の交代ですか？
        例）3/1

あなた：3/1

ボット：以下の内容で交代申請します。よろしいですか？

        📅 3月1日(日)
        👤 田中勇平 ↔ 星野竜大

        はい／いいえ

あなた：はい

ボット：✅ 交代が完了しました！

        📅 3月1日(日)
        👤 田中勇平　13:00〜20:00
        👤 星野竜大　10:00〜18:00
```

**注意**：交代はシフトシートに双方の勤務時間が登録されている場合のみ実行できます。

---

## 操作中のキャンセル方法

どのステップでも以下の方法でキャンセルできます。

- 変更の種類選択時に「**3**」または「**まちがえました**」と入力
- 変更確認時に「**いいえ**」と入力

```
あなた：いいえ

ボット：キャンセルしました。
```

---

## 申請期限

**変更対象日の前日 11:30まで**が申請期限です。

期限を過ぎると以下のメッセージが表示されます。

```
ボット：3月1日(日) の変更期限（前日11:30）を過ぎています。
        店長に直接ご連絡ください。
```

---

## 時刻の入力形式

| 形式 | 例 |
|------|----|
| HH:MM〜HH:MM | 15:30〜18:00 |
| HH:MM~HH:MM（半角チルダ） | 15:30~18:00 |
| HH:MM～HH:MM（全角波線） | 15:30～18:00 |
| HH:MM-HH:MM（ハイフン） | 15:30-18:00 |

---

## セッションタイムアウト

操作を開始してから **10分間** 返信がないと、セッションがリセットされます。
その場合は最初からやり直してください。

---

## シフト確認リマインド（ボットからの自動送信）

毎日12時台にグループへリマインドが届きます。

```
【シフトリマインド】3月1日(日) のシフト

10:00〜18:00　田中勇平　　[確認しました]
13:00〜20:00　星野竜大　　[確認しました]
15:00〜20:00　杉本尚哉　　[確認しました]
```

**【確認しました】ボタンを押すと**
- スプレッドシートの「確認状況」列に「確認済み」が記録される
- 17時台の追っかけリマインドの対象から外れる

**17時台の追っかけリマインド（未確認者のみ）**

```
【シフト未確認リマインド】
以下のスタッフの皆さん、明日のシフト確認をお願いします！

@田中勇平  15:30〜18:00

確認したら、12:00のメッセージの【確認しました】ボタンを押してください。
```

---

## シフト不足アラート（自動送信）

5日後の日付でシフトが不足している時間帯がある場合、自動でアラートが届きます。

```
🚨 【シフト不足のお知らせ】
3月5日(木) のシフトが不足しています！

不足時間帯: 10:00〜12:00, 15:00〜17:00
該当するスタッフの方はご協力をお願いします！
```

---

## エラーメッセージ一覧

| メッセージ | 原因 | 対処 |
|-----------|------|------|
| 名前が見つかりませんでした | 入力した名前がスタッフシートと一致しない | スペースなしのフルネームで再入力 |
| 自分自身とは交代できません | 交代相手に自分の名前を入力した | 別のスタッフ名を入力 |
| 〇月〇日 のシフトが見つかりませんでした | 対象日にシフトが未登録 | 店長に連絡 |
| 変更期限を過ぎています | 前日11:30を過ぎている | 店長に直接連絡 |
| 日付の形式が認識できませんでした | 入力形式が違う | 3/1 または 3月1日 の形式で入力 |
| 時間の形式が認識できませんでした | 入力形式が違う | 15:30〜18:00 の形式で入力 |
