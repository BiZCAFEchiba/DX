# スプレッドシート操作仕様書

## 概要

シフトリマインダーシステムはGoogleスプレッドシートをデータベースとして使用します。
メニュー「ShiftReminder」からほぼすべての操作が行えます。

---

## シート構成

| シート名 | 用途 |
|---------|------|
| シフト | スタッフのシフトデータ（メインデータ） |
| スタッフ | スタッフ名とLINE WORKS IDの対応表 |
| 送信ログ | リマインド送信の履歴 |
| 営業時間 | 曜日ごとの営業時間・休業日設定 |
| 設定 | 自動送信の実行時刻設定 |

---

## 各シートの説明

### シフトシート

スタッフのシフトデータを管理します。

| 列 | 列名 | 内容 | 例 |
|----|------|------|----|
| A | 日付 | シフト日付 | 2026/03/01 または 2026-03-01 |
| B | 曜日 | 曜日（自動入力） | 月 |
| C | スタッフ名 | スタッフのフルネーム | 田中勇平 |
| D | 開始時刻 | 勤務開始時刻 | 10:00 |
| E | 終了時刻 | 勤務終了時刻 | 18:00 |
| F | 業務内容 | 担当業務（任意） | フロア / キッチン |
| G | 登録日時 | 登録・更新日時（自動） | - |
| H | 登録元 | データの登録元（自動） | pdf_import / webhook_update |
| I | 確認状況 | リマインド確認済みフラグ | 確認済み |

**注意点**
- 1スタッフ1日1行が基本
- チャットからシフト変更があった場合、H列が `webhook_update` または `webhook_create` に更新される
- I列の「確認済み」はLINE WORKSで【確認しました】ボタンを押したときに自動記入される

---

### スタッフシート

LINE WORKSへのメッセージ送信に必要なID情報を管理します。

| 列 | 列名 | 内容 | 例 |
|----|------|------|----|
| A | スタッフ名 | フルネーム（シフトシートのC列と一致させること） | 田中勇平 |
| B | LINE WORKS ID（送信用） | リマインド送信に使うID（email形式） | shirucafe_XXXXXXXX@enrission-46cafe |
| C | 有効 | スタッフの有効フラグ（未使用） | - |
| D | 登録日 | 初回登録日（任意） | - |
| E | Googleカレンダー用メール | カレンダー招待用メール（任意） | - |
| F | 受信用ID（webhook UUID） | チャット受信・確認判定に使うUUID | 794d4bff-xxxx-xxxx-xxxx-xxxxxxxxxxxx |

**B列とF列の違い**

| | B列（送信用） | F列（受信用） |
|--|-------------|-------------|
| 形式 | email形式 | UUID形式 |
| 用途 | ボットからスタッフへのDM送信 | スタッフからの受信識別・確認判定 |
| 登録方法 | 手動 | **自動登録**（後述） |

**F列（受信用ID）の自動登録方法**

1. スプレッドシートのメニュー「ShiftReminder」→「受信ID取得」をクリック
2. ボットが全スタッフにDMを送信し「フルネームを返信してください」と案内
3. スタッフがLINE WORKSで自分のフルネームを返信
4. ボットが名前を照合し、F列にUUIDを自動書き込み

**受信用IDが登録されていない場合の影響**
- シフトリマインドの送信は可能（B列で送信）
- チャットからのシフト変更・照会が使えない
- 【確認しました】ボタンを押しても確認済みとして記録されない

---

### 営業時間シート

定休日・営業時間を設定します。シフト不足アラートの判定に使用されます。

| 列 | 内容 | 例 |
|----|------|----|
| A | 曜日 | 月 / 火 / 水 / 木 / 金 / 土 / 日 |
| B | 開店時間 | 10:00 |
| C | 閉店時間 | 20:00 |
| D | 営業日チェック | ☑（チェックあり = 営業日） |

**定休日の設定方法**
- D列のチェックを外す → その曜日は定休日扱い
- 日本の祝日は自動で休業日扱い（Googleカレンダー連携）

---

### 設定シート

自動送信の実行時刻を設定します。

| 設定項目 | デフォルト値 | 説明 |
|---------|------------|------|
| 翌日リマインド実行時間（時） | 12 | 毎日12時台にリマインドを送信 |
| 追っかけリマインド実行時間（時） | 17 | 未確認スタッフへの再送時刻 |
| シフト不足アラート実行時間（時） | 12 | 5日後のシフト不足チェック時刻 |

時刻を変更した場合は「ShiftReminder」→「自動送信（トリガー）設定」を再実行してください。

---

### 送信ログシート

リマインド送信の履歴が自動記録されます。

| 列 | 内容 |
|----|------|
| A | 送信日時 |
| B | 対象日 |
| C | スタッフ数 |
| D | 送信方法（auto / manual / webhook） |
| E | 結果（success / error / skip） |
| F | 詳細メモ |

**自動クリーンアップ**：100行を超えると古い行が自動削除され、最新50行のみ保持されます。

---

## メニュー操作一覧（ShiftReminder）

### シート初期化
全シート（シフト・スタッフ・送信ログ・営業時間・設定）を新規作成します。
> 既存シートがある場合は注意。確認ダイアログが2回表示されます。

### 「設定」シート作成
設定シートのみを個別に作成します。

### PDF解析 → シート取込
GoogleドライブのPDFフォルダからシフト表PDFを取り込みます。

**手順**
1. シフト表PDFをGoogleドライブの指定フォルダにアップロード
2. このメニューをクリック
3. PDFを自動解析し、シフトシートに追記
4. 処理済みPDFはゴミ箱に移動
5. 重複するデータはスキップされる

### リマインド手動送信
手動でリマインドをLINE WORKSグループに送信します。

- 12時前に実行 → 当日のシフトを送信
- 12時以降に実行 → 翌日以降の次営業日シフトを送信
- 確認ダイアログでスタッフ数を確認してから送信

### スタッフ更新
スタッフシートからIDマッピングを再読み込みします。スタッフを追加・変更した後に実行してください。

### 受信ID取得
全スタッフに個別DM（ダイレクトメッセージ）を送り、フルネームの返信を促します。返信があったスタッフのF列（受信用ID）が自動登録されます。

### 既存シフトをカレンダー同期
シフトシートのデータをGoogleカレンダーに登録します。

### 自動送信（トリガー）設定
設定シートの時刻に基づいて、自動送信トリガーを作成・更新します。
初回設定時および設定シートを変更したときに実行してください。

### 【テスト】1分後に追っかけ実行
テスト用。約1分後に追っかけリマインドを強制実行します。
テスト後は「自動送信（トリガー）設定」を再実行して通常スケジュールに戻してください。

---

## 自動送信の仕組み

| タイミング | 処理内容 |
|-----------|---------|
| 毎日12時台 | 次営業日のシフトをグループに一括送信。各スタッフに【確認しました】ボタン付き |
| 毎日17時台 | 未確認のスタッフにメンション付きで再送 |
| 毎日12時台 | 5日後のシフト不足をチェック。不足があればアラート送信 |

**フォールバック動作**
シフトシートにデータがない場合、GoogleドライブのPDFから自動解析を試みます。

---

## よくある操作フロー

### 新しいシフト期間の登録

1. シフト表PDFをGoogleドライブの指定フォルダにアップロード
2. 「PDF解析 → シート取込」を実行
3. シフトシートを確認し、誤りがあれば直接修正

### スタッフを追加するとき

1. スタッフシートのA列にフルネーム、B列に送信用IDを入力
2. 「スタッフ更新」を実行
3. 「受信ID取得」を実行してF列のUUIDを登録

### 手動でシフトを修正するとき

シフトシートのD列（開始時刻）・E列（終了時刻）を直接編集してください。
