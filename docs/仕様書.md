# シフトリマインドBot 仕様書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | BizCAFE シフトリマインドBot |
| 目的 | 翌日のシフトスタッフへLINE WORKSグループで自動リマインド通知を行う |
| 実行タイミング | 毎日 正午（12:00） |
| 対象 | BizCAFE千葉大学店 スタッフ |
| 技術基盤 | **Google Apps Script（完全無料・課金なし）** |

---

## 2. システム全体フロー

```
[毎日12:00 GASタイムトリガー起動]
        |
        v
[Google Drive] ── DriveApp でPDFファイル取得（GAS組み込み・無料）
        |
        v
[PDF → Googleドキュメント変換（OCR）] ── テキスト抽出
        |
        v
[翌日のシフト情報を解析] ── スタッフ名・時間帯を抽出
        |
        v
[スタッフ名 → LINE WORKS アカウント マッピング]
        |
        v
[LINE WORKS Bot API] ── UrlFetchApp でメンション付きリマインド送信
        |
        v
[Google Drive] ── DriveApp で処理済みPDFをゴミ箱に移動
```

---

## 3. なぜGoogle Apps Scriptか

| 比較項目 | Google Cloud API | Google Apps Script |
|----------|------------------|--------------------|
| 費用 | 課金制（API呼び出し課金） | **完全無料** |
| Google Drive操作 | Drive API v3（要課金設定） | **DriveApp（組み込み・無料）** |
| PDF解析 | 外部ライブラリ必要 | **Docs変換でOCRテキスト抽出（無料）** |
| スケジュール実行 | Cloud Scheduler（要課金設定） | **タイムトリガー（無料）** |
| サーバー管理 | Cloud Functions設定必要 | **不要（Google側で実行）** |
| 認証設定 | サービスアカウント発行 | **Googleアカウントのみ** |

## 4. 機能要件

### 4.1 PDF取得（Google Drive）

| 項目 | 内容 |
|------|------|
| 取得元 | Google Drive 指定フォルダ |
| ファイル形式 | PDF |
| 取得方法 | `DriveApp.getFolderById()` → `getFilesByType()` |
| 認証 | GASスクリプト所有者のGoogleアカウント権限 |
| 対象ファイル | 指定フォルダ内のPDFファイル |

- フォルダIDをスクリプト内定数で設定
- フォルダ内にPDFが存在しない場合はスキップ（ログ記録）

### 4.2 PDF解析・シフトデータ抽出

| 項目 | 内容 |
|------|------|
| 解析方法 | PDFをGoogleドキュメントに変換（OCR） |
| 使用API | `Drive.Files.insert()`（Advanced Drive Service・無料） |
| 抽出対象 | 翌日（実行日+1日）のシフト情報 |
| 抽出データ | 日付、スタッフ名、シフト時間帯、業務内容 |

#### 実際のPDFフォーマット（確認済み）

```
423BiZCAFE（千葉大学）店
2026年02月09日(月)
スタッフ  開始・終了時刻  12  13  14  15  16  17  18
星野竜大   15:30〜17:30
杉本尚哉   13:30〜15:30
                                              計:4時間0分
2026年02月10日(火)
スタッフ  開始・終了時刻  12  13  14  15  16  17  18
杉本尚哉   13:45〜15:30
武田祥哉   15:30〜17:30
                                              計:3時間45分
```

#### PDFの特徴
- **1つのPDFに複数日分**のシフトを含む（日数は不定：数日〜1週間以上の場合あり）
- 祝日・休業日は記載なし（スキップされる）
- 業務内容の記載あり（例：`清掃`、`在報・棚卸`）
- 日付フォーマット：`2026年MM月DD日(曜日)`
- 時刻フォーマット：`HH:MM〜HH:MM`（全角チルダ `〜`）
- ファイル名は不定（日付範囲 `9-13.pdf` 等だが、命名規則に依存しない設計とする）
- **PDF内の日付ブロック数は可変** → 日付の正規表現マッチで柔軟に分割

#### 解析で使用する正規表現

| 対象 | 正規表現パターン | 説明 |
|------|------------------|------|
| 日付行 | `(\d{4})年(\d{2})月(\d{2})日\((.)\)` | 日付ブロックの区切りを検出 |
| スタッフ行 | `(.+?)\s+(\d{1,2}:\d{2})[〜~](\d{1,2}:\d{2})` | 名前と勤務時間を抽出 |
| 業務内容 | スタッフ行の時刻以降に続くテキスト | `清掃`、`在報・棚卸` 等を補注として抽出 |

#### 処理手順
1. `DriveApp` でPDFファイルの Blob を取得
2. `Drive.Files.insert()` でPDF → Googleドキュメントに変換（OCR有効）
3. `DocumentApp.openById()` でテキスト全文を取得
4. テキストを日付ブロックごとに分割
5. 翌日の日付ブロックからスタッフ名・時間帯を正規表現で抽出
6. 変換用の一時Googleドキュメントを削除

- 解析失敗時は管理者にLINE WORKS通知

#### スタッフ管理方針
- PDFに記載されるスタッフ名はそのまま表示される（マッピング不要でも動作）
- LINE WORKSメンションを有効にするには `config.gs` のマッピングにアカウントIDを追記
- **スタッフの増減に応じて `config.gs` を更新するだけ**で対応可能
- マッピング未登録のスタッフが検出された場合、ログに警告を出力

### 4.3 LINE WORKS メンション付きリマインド送信

| 項目 | 内容 |
|------|------|
| 送信先 | LINE WORKS 指定グループ |
| API | LINE WORKS Bot API 2.0 |
| 認証方式 | Service Account (JWT) |
| HTTP送信 | `UrlFetchApp.fetch()`（GAS組み込み） |
| メッセージ形式 | テキスト（メンション付き） |

#### メッセージ例

**業務内容なしの場合：**
```
【明日のシフトリマインド】
📅 2026年2月10日（火）

@杉本尚哉  13:45〜15:30
@武田祥哉  15:30〜17:30

明日もよろしくお願いします！
出勤時間の確認をお願いします。
```

**業務内容ありの場合（補注付き）：**
```
【明日のシフトリマインド】
📅 2026年2月13日（金）

@田中勇平  13:45〜15:30
  └ 📋 在報・棚卸
@杉本尚哉  15:30〜18:00
  └ 📋 在報・棚卸 / 清掃
@星野竜大  17:00〜18:00
  └ 📋 清掃

明日もよろしくお願いします！
出勤時間の確認をお願いします。
```

#### メッセージ仕様
- シフト開始時間の早い順に表示
- 業務内容がある場合のみ `└ 📋` 付きで補注行を追加（ない場合は省略）
- 1人に複数業務がある場合は ` / ` 区切りで表示

#### メンション仕様
- スタッフ名とLINE WORKSアカウントIDの対応表をスクリプト内で管理
- `mentionedList` パラメータでユーザーIDを指定しメンション実現
- マッピングに該当しないスタッフ名は名前のみ表示（メンションなし）
- **新規スタッフはマッピング未登録でもシフト表示される**（メンションのみ無効）

### 4.4 処理済みPDFのゴミ箱移動

| 項目 | 内容 |
|------|------|
| 対象 | 解析・送信が正常完了したPDFファイル |
| 処理 | `file.setTrashed(true)`（DriveApp組み込み） |
| タイミング | リマインド送信成功後 |

- 送信失敗時はゴミ箱に移動しない（再処理可能にする）

### 4.5 シフト変更反映（前日11:30締め切り）

| 項目 | 内容 |
|------|------|
| 目的 | シフト前日 11:30 までに提出された変更依頼を自動的に反映する |
| 受付方法 | 専用の「シフト変更フォーマット」シート（Googleスプレッドシート） |
| 反映対象 | シフトの延長 / 一部または全てを「ある人からある人へ」変更する依頼 |
| 反映タイミング | リマインド送信の直前（前日 11:30 以降の実行で確定） |

- 変更フォーマットには以下のような項目を持つ想定：
  - 日付
  - 元のスタッフ名
  - 変更後スタッフ名
  - 変更前シフト時間帯（開始〜終了）
  - 変更後シフト時間帯（開始〜終了）
  - 依頼種類（延長 / 一部変更 / 全変更 など）
- GAS側では：
  - PDFから抽出した翌日のシフトデータに対し、
  - 前日 11:30 時点で「有効な変更依頼」を順に適用し、
  - 最終的なシフト一覧を LINE WORKS に送信する。
- 11:30 を過ぎてからの変更は自動反映対象外（運用ルールで制御）。

### 4.6 営業時間不足検知と @all 通知

| 項目 | 内容 |
|------|------|
| 目的 | 営業時間内にスタッフがいない時間帯を事前に検知し、早めに募集をかける |
| 対象 | 5日後の営業日のシフト |
| 通知タイミング | 不足発生日の 5日前の 10:00 |
| 通知方法 | LINE WORKSグループに `@all` メンション付きで不足時間を通知 |

- 営業時間の中で、どの時間帯にもシフトが入っていない時間を「不足時間」と定義。
- 不足検知ロジック（例）：
  - 営業時間を 30 分単位でスロット化
  - 各スロットに誰か1人以上のシフトが割り当たっているかをチェック
  - 誰もいないスロットが連続する区間を「不足時間」として抽出
- 通知メッセージ例：
  - `5日後（YYYY/MM/DD）のシフトに不足があります。`
  - `不足時間： 13:00〜14:30`
  - `@all シフト調整をお願いします。`
- 不足時間がない場合は通知を行わない。

### 4.7 営業時間の設定（スプレッドシート管理）

| 項目 | 内容 |
|------|------|
| 目的 | ターム（学期・期間）ごとに変わる営業時間を柔軟に管理する |
| 管理場所 | Googleスプレッドシート（「営業時間設定」シート） |
| 単位 | 日付ごとの営業時間帯（開始〜終了） |

- 営業時間設定シートのイメージ：
  - 列：`日付` / `開始時刻` / `終了時刻` / `メモ（任意）`
  - 例：`2026-04-01, 11:00, 19:00`
- タームごとに営業時間が変わる場合：
  - 期間単位で複数行を登録することで対応。
  - 指定日付に一致する行がない場合は「休業日」とみなす（シフト送信・不足検知をスキップ）。
- GAS側の利用箇所：
  - 翌日のシフトリマインド送信時：該当日の営業時間を取得し、その範囲内でシフト時間を表示。
  - 5日前の不足検知時：該当日の営業時間をもとに「不足時間」を計算。
- 営業時間の更新は、原則として店長または管理者がスプレッドシート上で行う運用とする。

## 8. 処理詳細フロー

```javascript
function main() {
  // 1. Google Driveの指定フォルダからPDF一覧を取得
  const pdfFiles = getShiftPdfsFromDrive(DRIVE_FOLDER_ID);
  if (pdfFiles.length === 0) {
    Logger.log('PDFファイルが見つかりません。スキップします。');
    return;
  }

  const tomorrow = getTomorrow();
  let shiftFound = false;

  // 2. 各PDFを順に処理（翌日のシフトが見つかるまで）
  for (const pdfFile of pdfFiles) {
    // 3. PDFをGoogleドキュメントに変換してテキスト抽出
    const shiftText = extractTextFromPdf(pdfFile);
    if (!shiftText) {
      Logger.log('PDF解析失敗: ' + pdfFile.getName());
      continue;
    }

    // 4. 翌日のシフト情報を解析（業務内容も含む）
    // shiftData = [{ name: '杉本尚哉', start: '15:30', end: '18:00', tasks: ['在報・棚卸', '清掃'] }, ...]
    const shiftData = parseShiftData(shiftText, tomorrow);
    if (shiftData.length === 0) continue;

    // 5. LINE WORKSにメンション付きリマインド送信（業務補注付き）
    const success = sendLineWorksReminder(shiftData, tomorrow);

    // 6. 送信成功時のみPDFをゴミ箱に移動
    if (success) {
      pdfFile.setTrashed(true);
      Logger.log('処理完了：' + pdfFile.getName() + ' をゴミ箱に移動しました。');
    }
    shiftFound = true;
    break;
  }

  if (!shiftFound) {
    Logger.log('翌日（' + tomorrow + '）のシフト情報が見つかりませんでした。');
  }
}
```

## 12. 注意事項

- PDFのフォーマットが変更された場合、`pdfParser.gs` の正規表現修正が必要
- スタッフの入退社時に `config.gs` のマッピング更新が必要
- LINE WORKS API のアクセストークンは有効期限あり → JWTで都度発行（スクリプト内で自動処理）
- GASの実行ログは「実行数」メニューから確認可能
- 祝日・休業日のスキップ機能は Phase 1 では未実装（必要に応じて追加）
- OCRの精度はPDFの品質に依存 → テキストベースPDFなら高精度、画像PDFの場合は要調整
